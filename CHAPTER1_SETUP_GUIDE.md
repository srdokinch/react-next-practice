# 進め方の1章 手順書（プロジェクト準備）

この章は「Next.js + Tailwindのプロジェクトを作り、開発サーバが動く状態」にすることがゴールです。  
はじめての人でも迷わないように、作業の順番と確認ポイントを細かく分けています。

---

## ゴール
- Next.js（App Router）+ Tailwindのプロジェクトが作成できている
- `app/`配下の構成が確認できている
- 開発サーバが起動し、ブラウザで初期画面が表示できる

---

## 事前確認（作業前に1回だけ）
1. **Node.jsが入っているか確認**  
   ターミナルで以下を実行して、バージョンが表示されればOKです。  
   - `node -v`
   - `npm -v`

2. **作業用フォルダを決める**  
   例: `~/Git/` のような開発用フォルダにプロジェクトを作成します。

---

## 手順
### 1. プロジェクトを作成する
1. ターミナルで作業フォルダへ移動する  
2. `npx create-next-app@latest プロジェクト名` を実行する  
   - 例: `npx create-next-app@latest my-pomodoro-app`
   - プロジェクト名は任意（例: `pomodoro-timer` など）
3. 対話形式で質問されるので、以下に合わせて選択する  
   - TypeScript: 無効（JavaScriptを使用）  
   - App Router: 有効  
   - Tailwind CSS: 有効  
   - ESLint: 有効  
   - `src/`ディレクトリ: どちらでもOK（迷うなら無効）

**確認ポイント**  
作成が終わると、指定したプロジェクト名のフォルダができている。

---

### 2. プロジェクトの構成を確認する
1. プロジェクトフォルダへ移動する  
2. `app/`フォルダがあることを確認する  
3. `app/page.js` と `app/globals.css` が存在することを確認する

**確認ポイント**  
上記のファイルが見つかればOK。見つからない場合は、作成時の設定を確認する。

---

### 3. 開発サーバを起動する
1. `npm run dev` を実行する  
2. 表示されたURL（通常 `http://localhost:3000`）をブラウザで開く

**確認ポイント**  
ブラウザにNext.jsの初期画面が表示されればOK。

---

## よくあるつまずき
- **`command not found: create-next-app` と出る**  
  → `npx` を付けて実行してください。正しくは `npx create-next-app@latest プロジェクト名` です。

- **node/npmのバージョンが出ない**  
  → Node.jsのインストールが必要です。

- **`app/`が見当たらない**  
  → App Routerを有効にして作成したかを確認してください。

- **`npm run dev`が失敗する**  
  → 依存インストールが途中で止まっていないか確認してください。

---
